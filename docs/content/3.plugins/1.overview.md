# Plugins

Nuxt Upload Kit uses a plugin architecture that allows you to extend and customize every aspect of the upload process.

## Plugin Types

There are two types of plugins:

### Processing Plugins

Processing plugins handle file transformation and custom validation:

- **Processors** - Transform files (compression, thumbnails)
- **Custom validators** - File checks that go beyond built-in restrictions (e.g., virus scanning, content inspection)

```ts
import { PluginImageCompressor } from "nuxt-upload-kit"

const uploader = useUploadKit({
  plugins: [PluginImageCompressor({ quality: 0.8 })],
})
```

::prose-tip
For common rules like file size, file count, and allowed MIME types, use [`uploadKit.restrictions`](/get-started/configuration#restrictions) in `nuxt.config.ts`. Restrictions are enforced on both the client and the server automatically — no plugin needed.
::

### Storage Plugins

Storage plugins handle uploading, downloading, and deleting files from remote storage. They are registered in `server/upload.server.config.ts`:

```ts [server/upload.server.config.ts]
import { defineUploadServerConfig, PluginAzureDataLake } from "nuxt-upload-kit/server"

export default defineUploadServerConfig({
  storage: PluginAzureDataLake({
    sasURL: process.env.AZURE_SAS_URL!,
    path: "uploads",
  }),
})
```

::prose-tip
Only one storage plugin can be active. The mode (`presigned` vs `server`) is derived from the storage adapter at startup. To pin it explicitly, set `mode: "server"` on `defineUploadServerConfig`.
::

## Built-in Plugins

### Processors

| Plugin                     | Description                     |
| -------------------------- | ------------------------------- |
| `PluginThumbnailGenerator` | Generates preview thumbnails    |
| `PluginImageCompressor`    | Compresses images before upload |
| `PluginVideoCompressor`    | Compresses videos using FFmpeg  |

### Storage

| Plugin                | Description             |
| --------------------- | ----------------------- |
| `PluginAzureDataLake` | Azure Data Lake Storage |

## Plugin Lifecycle

Plugins hook into different stages of the file lifecycle:

```
File Selected
     ↓
┌─────────────────────┐
│  1. RESTRICTIONS    │  ← uploadKit.restrictions (size, count, mime)
│     Built-in checks │
│     run first       │
└─────────────────────┘
     ↓
┌─────────────────────┐
│  2. VALIDATE        │  ← Custom processing plugins with a `validate` hook
│     Check if file   │
│     should be added │
└─────────────────────┘
     ↓
┌─────────────────────┐
│  3. PREPROCESS      │  ← PluginThumbnailGenerator
│     Immediate UI    │
│     preparation     │
└─────────────────────┘
     ↓
   File Added to List
     ↓
   User Clicks Upload
     ↓
┌─────────────────────┐
│  4. PROCESS         │  ← PluginImageCompressor, PluginVideoCompressor
│     Transform file  │
│     before upload   │
└─────────────────────┘
     ↓
┌─────────────────────┐
│  5. UPLOAD          │  ← Storage adapter (server-side)
│     Send to server  │
└─────────────────────┘
     ↓
┌─────────────────────┐
│  6. COMPLETE        │  ← Post-upload processing
│     Cleanup, notify │
└─────────────────────┘
```

## Adding Processing Plugins

```ts
import { PluginImageCompressor, PluginThumbnailGenerator } from "nuxt-upload-kit"

const uploader = useUploadKit({
  plugins: [
    PluginThumbnailGenerator(),
    PluginImageCompressor({
      maxWidth: 2048,
      quality: 0.8,
    }),
  ],
})
```

## Plugin Execution Order

Processing plugins execute in the order they're added. Each processor receives the output of the previous one.

## Runtime Plugin Addition

Add plugins after initialization:

```ts
const uploader = useUploadKit()

// Add based on user preference
if (userPreferences.compressImages) {
  uploader.addPlugin(PluginImageCompressor({ quality: 0.8 }))
}
```
